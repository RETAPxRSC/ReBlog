运算符重载让自定义类型（类/结构体）可以像内置类型一样使用 `+ - * / == << >> [] ()` 等运算符，本质上是**函数重载的一种形式**。

---

## x.1 基本概念

- 使用 `operator` 关键字为类定义新的运算符行为  
- 不能改变运算符的优先级、结合性、参数个数  
- 不能创造新的运算符  
- 不能重载：`.`、`.*`、`::`、`?:`、`sizeof`  
- 运算符重载本质上是一个函数：  
  - 成员函数：`A operator+(const A &other) const`  
  - 全局函数：`A operator+(const A &a, const A &b)`  

---

## x.2 成员函数 vs 全局函数

### - 1. 成员函数形式

```cpp
class A
{
public:
    A operator+(const A &other) const;
};
```

特点：

- 左操作数是当前对象 `this`  
- 只能控制右操作数  
- 常用于 `+ - * / [] ()` 等  

### - 2. 全局函数形式

```cpp
A operator+(const A &a, const A &b);
```

特点：

- 左右操作数都可自定义  
- 常用于 `<<`、`>>`、` == ` 、` != ` 等  
- 当左操作数不是类对象时必须使用全局（如 `cout << obj`）

## x.3 常见运算符重载示例

### ① `+` 加法运算符

```cpp
class Point
{
public:
    int x, y;

    Point operator+(const Point &other) const
    {
        Point temp;
        temp.x = x + other.x;
        temp.y = y + other.y;
        return temp;
    }
};
```

---

### ② ` == ` 比较运算符

```cpp
bool operator==(const Point &a, const Point &b)
{
    return a.x == b.x && a.y == b.y;
}
```

---

### ③ `<<` 输出运算符（必须全局）

```cpp
ostream& operator<<(ostream &out, const Point &p)
{
    out << "(" << p.x << ", " << p.y << ")";
    return out;
}
```

---

### ④ `[]` 下标运算符

```cpp
int& operator[](int index)
{
    return arr[index];
}
```

---

### ⑤ `()` 函数调用运算符（仿函数）

```cpp
int operator()(int a, int b)
{
    return a + b;
}
```

---

### ⑥ ` = ` 赋值运算符（深拷贝）

```cpp
A& operator=(const A &other)
{
    if (this == &other) return *this;
    // 深拷贝逻辑
    return *this;
}
```

---

## x.4 为什么很多运算符要返回引用？

常见于：

- ` = `  
- `+=`  
- `[]`  
- `<<`  

原因：

- 支持链式操作  
- 避免不必要的拷贝  
- 保持行为与内置类型一致  

示例：

```cpp
a = b = c;
cout << a << b << c;
```

---

## x.5 const 的使用规范

- 参数通常使用 `const &`  
- 成员函数通常加 `const`  
- 返回值一般不加 `const`（会破坏链式操作）  

示例：

```cpp
A operator+(const A &other) const;
```

---

## x.6 友元与运算符重载

当运算符需要访问类的私有成员且不能写成成员函数时（如 `<<`），使用 `friend`：

```cpp
class Point {
    friend ostream& operator<<(ostream&, const Point&);
};
```

## x.7 运算符重载的底层本质（你未来一定会用到）

- 运算符重载只是语法糖，本质是函数调用  
- 编译器会把 `a + b` 转换为 `a.operator+(b)` 或 `operator+(a, b)`  
- 重载运算符不会自动处理深拷贝，需要你自己写  
- STL 容器、迭代器、智能指针大量依赖运算符重载  

未来：
- 自定义容器  
- 数学向量类  
- 字符串类  
- 智能指针  
- 迭代器  
都会大量使用运算符重载。

---

## x.8 推荐的运算符重载写法（你未来一定会用到）

### - 1. 所有二元运算符都应使用 `const &` 参数  
### - 2. 所有不修改对象的运算符都应加 `const`  
### - 3. `<<` 和 `>>` 必须写成全局 + 友元  
### - 4. ` = ` 必须处理自赋值  
### - 5. `[]` 必须返回引用  
### - 6. `()` 可用于实现仿函数（函数对象）
